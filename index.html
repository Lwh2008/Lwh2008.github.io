5最简单暴力的方法

入门 UE5 的步骤：

  1. 安装 UE5：
    • 访问 Unreal Engine 的官方网站。
    • 注册并下载 UE5。
    • 根据操作系统安装 UE5。
  2. 了解界面：
    • 启动 UE5，熟悉编辑器的界面布局。
    • 学习如何使用场景视图、内容浏览器、工具栏、世界大纲等。
  3. 创建新项目：
    • 选择 “文件” -> “新建项目”。
    • 选择一个模板，设置项目名称和保存位置。
    • 配置项目设置，然后点击 “创建”。
  4. 学习基础：
    • 学习如何导入资产（如模型、纹理）。
    • 学习基本的场景构图，包括放置对象、调整变换（位置、旋转、缩放）。
    • 了解材质和纹理的基础知识，学习如何创建和应用材质。
  5. 探索核心特性：
    • 学习 Nanite 的使用，它可以让你导入电影级别的几何细节，而无需担心多边形数量。
    • 学习 Lumen，这是一个全动态全局光照解决方案，可以实时地响应场景中的变化。
  6. 尝试蓝图可视化脚本：
    • 蓝图是 UE5 中的一个可视化脚本系统，允许开发者无需编写代码就能创建逻辑。
    • 学习如何使用蓝图创建简单的游戏机制，如角色移动、摄像机控制等。
  7. 实践项目：
    • 开始一个小项目，实践你学到的知识。
    • 尝试构建一个简单的游戏场景，或者复刻一个你喜欢的游戏的一部分。
  8. 学习资源：
    • 利用 Epic Games 提供的官方文档和教程。
    • 观看教学视频，参加在线课程和论坛。
  9. 加入社区：
    • 加入 UE5 的开发者社区，如 Reddit、Stack Overflow、官方论坛等。
    • 参与讨论，获取帮助，分享你的项目和经验。
  10. 持续学习：
    • UE5 是一个复杂的工具，需要时间和实践来掌握。
    • 持续学习新的功能和最佳实践，不断改进你的技能。
      UE5 的学习曲线可能会比较陡峭。

6.Docker入门

Docker是一个现代化的容器引擎，它适用多个平台，适用于Linux，Windows，macos，他原先仅适用于Linux平台，是为了更方便的去配置软件和服务。 由于他的所有容器里头的环境都是共享的系统主机内核，所以说他一般情况下不需要虚拟化，从而提高了运行的效率和性能。而且它的容器仅仅包含某个特定系统的依赖，而且这些环境仅仅存在的是依赖，没有其他的，所以说大小一般情况下在50MB左右。 而且由于它是一个引擎且适用于多平台，所以说打包好了一个容器可以在其他平台的docker上使用。只要对方平台支持该引擎，那它就能成功运行。
Docker安装，注意，这里用的是官方给的建议选项，就是使用官方的脚本
curl -fsSL https://get.docker.com -o get-docker.sh

搜索镜像

启动Docker镜像（以启动Debian的Docker环境为例）

注：如果要给某一个Docker-runtime命名，可以加上–name name，比如说在上述命令里面命名当前Debian容器为Debian

sudo docker run -it --name debian debian /bin/bash

退出当前环境只需要在容器里面输入exit即可，但是由于docker是运行时环境，每次退出时如果再用相同的命令再启动的话，它不会回到原来的环境，而是直接会打开一个新的实例。如果要回到之前的容器只需要启动之前的容器即可

sudo docker start debian

此时是基于前面已经将那个环境命名为Debian的情况下，回到环境（回到了环境只能是已经启动的环境，已经退出了环境必须先启动再回到

sudo docker attach debian

一般情况下，容器在未名的状态下，他们是有容器的ID，容器ID可以用它来查看

sudo docker ps -a

↑列出开启的所有Docker容器

清理系统里面运行所有Docker容器

sudo docker system prune

清理某一个容器

sudo docker rm -f name

导出容器

sudo docker export name > name.tar

卸载 docker

sudo apt purge docker-ce

删除镜像、容器、配置文件等内容

sudo rm -rf /var/lib/docker

5.apt使用

Debian里面的包管理器(apt)基本用法↓

注：需要以root权限运行

安装软件

sudo apt install name

注：name代表软件名

移除软件（保留配置文件）

sudo apt remove name

移除软件（删除软件和配置文件）

sudo apt remove --purge name

检查软件更新（仅仅是检查软件更新和更新源）

sudo apt update

修复系统依赖

sudo apt install -f

查看系统里面所有安装的软件

sudo apt list

注意，apt每次只允许运行一个进程，如果还要运行新的apt进程的话，它会提示进程被锁定，所以说此时你是要杀掉他的所有进程，使用sudo killall dpkg即可

当你杀掉了进程以后，你想再使用包管理器，它会提示你软件包未配置，使用

sudo dpkg --configure -a

有的时候系统里头会残留一些曾经需要用的，而现在用不着的依赖包，可以使用

sudo apt autoremove

自动移除用不着的依赖

有的时候在包管理器下载的包它会有残留，你可以使用

sudo apt clean all

以上内容虽然简短，但是日常使用肯定够用

其次，如果要一次性安装多个软件，只需要在后面依次输入软件名称就行，每个软件之间用空格隔开，类似于

sudo apt install a b c

当你安装或者是进行软件更新以及卸载软件的时候，他会问你是否要进行下一步操作。他默认情况下是选的是可以按下回车键来执行，或者是可以加上-y来直接执行下一步而无需提问

4.Markdown入门

Markdown 是一种轻量级标记语言，由 John Gruber 于 2004 年创建，现在已经成为世界上最受欢迎的标记语言之一。Markdown 的设计目标是易读易写，可以让作者以纯文本格式编写文档，然后转换成结构化的 HTML 文档。
Markdown 的基本语法包括以下几部分：

标题

在文字前面加上 # 来创建标题。# 的数量代表标题的级别。

段落和换行

直接编写文字就会创建段落。如果想在段落内换行，需要在行尾加上两个空格然后回车。

强调

  • 斜体 或 _斜体_：将文字两边各加上一个 * 或 _。
  • 粗体 或 __粗体__：将文字两边各加上两个 * 或 _。
  • 粗斜体 或 ___粗斜体___：将文字两边各加上三个 * 或 _。

列表

  • 无序列表使用 *、+ 或 - 作为列表标记：
  • 有序列表使用数字后跟英文句点：

链接和图片

  • 链接：[链接文字](URL "标题")
  • 图片：![替代文字](图片链接 "标题")

引用

在引用的文字前加上 >：

代码

  • 行内代码：用一对反引号包围代码 code。
  • 代码块：用三个反引号包围代码块，或者用四个空格缩进。 这是代码块

分隔线

使用三个以上的 *、- 或 _ 来创建分隔线：

这些是 Markdown 的基本使用方法，通过这些简单的标记，你可以编写出结构化的文档。Markdown 解析器会将这些标记转换为相应的 HTML 标签，从而在网页上展示出格式化的文本。

3.vi简单入门

虽然说大部分发行版默认编辑器不是vi而是nano，但是他的通用性很好，在Linux和FreeBSD等等系统上都可以用。

这里介绍他最基本的使用方法↓

首先，你要确定你的系统有没有安装vi

vi -v

如果有反应，并且他打开了一个显示版本号的页面，你可以输“：q”退出，如果报错提示“command not found”就请安装它（以Debian为例）

sudo apt install vim -y

使用方法↓vim默认情况下vim是命令模式，按下“i”进入编辑模式以后你就可以像普通的文本编辑器编辑文本。保存、退出或者是保存后退出或者是强制退出需要进入命令模式，从编辑模式退出到命令模式按下“esc”退出。

命令模式下输入退出vim“：q”（如果你对文本做了更改他会报错）

保存更改输入“：w”保存更改并退出输入“：wq”

强制退出不保留更改输入“：q！”

请注意，仅仅适用于最基本的使用，仅仅适用于日常编辑文本的工作。提醒一句，如果你要查看重要文本，而不想对他做出任何的整改，防患于未然，请用view。

2.Git

Git 是一个开源的分布式版本控制系统，由 Linus Torvalds 创造，用于帮助多人协作进行软件开发。Git 可以处理任何大小的项目，速度都非常快，并且具有灵活的分支和合并模型，这使 Git 成为了世界上最先进的版本控制系统。
Git 的工作流程通常涉及以下几个基本概念：

  1. 仓库（Repository）：存放代码的地方，分为本地仓库和远程仓库。
  2. 提交（Commit）：对代码库的更改，每次提交都会记录更改的作者、日期和描述。
  3. 分支（Branch）：从主开发线上分出的独立线路，用于开发新功能或修复问题。
  4. 合并（Merge）：将两个或多个分支的更改整合到一起。
  5. 拉取（Pull）：从远程仓库获取最新的版本并合并到本地分支。
  6. 推送（Push）：将本地的提交发送到远程仓库。
    使用 Git 需要安装 Git 客户端。安装后，可以通过命令行进行操作，例如：
  • git clone <url>：克隆一个远程仓库到本地。
  • git add <file>：将文件更改添加到暂存区。
  • git commit -m "message"：创建一个新的提交。
  • git push origin <branch>：将本地分支的更改推送到远程仓库。
  • git pull origin <branch>：从远程仓库拉取并合并到本地分支。
